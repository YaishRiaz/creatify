Fix two issues in creatify-web:

ISSUE 1 — Replace ALL .single() calls on 
creator_profiles, brand_profiles, and users 
tables with .maybeSingle()

.single() throws a 406 error when no row found
.maybeSingle() returns null instead — safe

Search entire codebase for:
.single()

Replace every instance that queries these tables:
- creator_profiles
- brand_profiles  
- users

With:
.maybeSingle()

Example:
BEFORE: .eq('user_id', session.user.id).single()
AFTER:  .eq('user_id', session.user.id).maybeSingle()

ISSUE 2 — Multiple GoTrueClient instances

Add this to the top of each layout file and 
the login page to prevent multiple instances.

In src/app/creator/layout.tsx at the top 
of the useEffect, add a ref guard:

Add state: const [initialized, setInitialized] = useState(false)

Wrap the useEffect body:
useEffect(() => {
  if (initialized) return
  setInitialized(true)
  
  // ... rest of auth check
}, [])

Do the same in brand/layout.tsx and admin/layout.tsx

ISSUE 3 — Failed to load profile

  .maybeSingle()  // ← use maybeSingle not single

// If no profile exists create one
